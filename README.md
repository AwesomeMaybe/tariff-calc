# Тарифный калькулятор УК

Веб-приложение для расчёта тарифа на содержание и управление многоквартирным домом (МКД) на основе технико-экономических показателей объекта и ежемесячных эксплуатационных затрат.

## Проблема

Управляющая компания обслуживает портфель жилых комплексов. Для каждого объекта нужно обосновать тариф (рубли за м² в месяц), исходя из реальной структуры затрат: ФОТ персонала, материалы, техническое обслуживание инженерных систем, клининг, охрана, благоустройство и т. д. Эти расчёты обычно живут в разрозненных Excel-таблицах: их тяжело поддерживать, сравнивать между объектами и пересчитывать при изменении наценки или НДС.

Калькулятор сводит расчёт в единый интерфейс:

- описывается паспорт объекта (площади, секции, лифты, инженерия, благоустройство);
- вносятся ежемесячные затраты по 19 эксплуатационным разделам;
- тариф пересчитывается автоматически — базовый и итоговый (с учётом прибыли и НДС);
- объекты сохраняются и сравниваются между собой, формируется сводка по всему портфелю УК.

## Возможности

- **Паспорт объекта (ТЭП)** — ввод площадей (жилые, нежилые, кладовые, машиноместа) и количеств помещений (квартиры, нежилые, кладовые, машиноместа), групп секций и этажности, групп лифтов, входных групп, инженерных систем (ВРУ, ИТП, вентиляция, водоподготовка), парковочной инфраструктуры и благоустройства территории.
- **Изометрическая модель МКД** — параметры объекта визуализируются в реальном времени в виде 2.5D-схемы (башни по секциям и этажности с остеклением, оборудование на кровле, подземный контур ВРУ/ИТП/паркинга, благоустройство двора).
- **Растягиваемые панели** — левый (объекты) и правый (сводка) бары тянутся за разделитель, ширина сохраняется локально.
- **19 разделов затрат** — управление, электрика, водоснабжение, ИТП, АДС, конструктив, лифты, вентиляция, пожарная безопасность, диспетчеризация, фильтрация воды, тепловые завесы, клининг, благоустройство, альпинистские работы, охрана, консьерж, технадзор, доп. клининг. Каждый раздел содержит редактируемые статьи (ФОТ, материалы, ТО и пр.).
- **Структура затрат** — визуализация долей разделов в общем бюджете и подсчёт активных разделов.
- **Автоматический расчёт тарифа** — базовый тариф (затраты / площадь) и итоговый тариф с применением коэффициентов прибыли и НДС, по каждому разделу и в целом по объекту.
- **Индексация тарифа** — годовой коэффициент индексации с проекцией итогового тарифа на 3 года вперёд (компаундинг) — стандартная практика для долгосрочных договоров УК.
- **Несколько объектов** — создание, переименование и удаление объектов; переключение между ними; автосохранение изменений.
- **Сводка по УК** — агрегированная статистика по всему портфелю (количество объектов, суммарная площадь, суммарные затраты).
- **Локальное хранение** — все данные сохраняются в браузере (localStorage), включая миграцию старых форматов данных.
- **Локализация** — интерфейс и форматирование чисел/валюты на русском языке (`ru-RU`, рубли).

## Логика расчёта

Расчёт реализован в `src/lib/calculate.ts`:

```
totalArea   = жилые + нежилые + кладовые + машиноместа
markup      = profitCoef × vatCoef            // множитель прибыли и НДС
tariffBase  = затраты_раздела / totalArea     // базовый тариф, р/м²
tariffFinal = tariffBase × markup             // итоговый тариф, р/м²
tariff(год) = tariffFinal × indexationCoef^год // индексация на N лет вперёд
```

Затраты раздела — сумма всех его статей (`monthly`). Итоговые показатели по объекту считаются суммированием затрат всех разделов и делением на общую площадь.

**Входы:** параметры объекта (`BuildingParams`) и затраты по разделам (`CostSection[]`).
**Выходы:** итоговая площадь, тариф по каждому разделу (базовый и итоговый), общий месячный бюджет и итоговый тариф по объекту (`CalcOutput`).

## Запуск

Требуется Node.js (LTS) и npm.

```bash
# установка зависимостей
npm install

# режим разработки (http://localhost:3000)
npm run dev

# production-сборка
npm run build

# запуск собранного приложения
npm run start
```

Для доступа с других устройств своего tailnet (приватно, без публичности) — `start.bat`: поднимает dev и пробрасывает порт через `tailscale serve` (не `funnel`).

## Оформление

Интерфейс выдержан в фирменном стиле **MR Group**: светлая премиум-тема (обсидиан + белый, фиолетовый акцент), шрифты **Inter** (текст) и **Unbounded** (заголовки/лого, широкий геометрический гротеск с кириллицей). Цвета вынесены в CSS-токены (`globals.css`), что позволяет менять всю палитру из одного места.

```
├── lib/
│   ├── calculate.ts            # формулы расчёта тарифа, индексация, агрегаты

```

Конфигурация: `next.config.ts`, `tailwind.config.ts`, `postcss.config.mjs`, `tsconfig.json`.
